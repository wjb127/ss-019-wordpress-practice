확인 방법 — 순서대로 체크
1
URL 확인
사이트 주소를 봤을 때 .wordpress.com이 붙어있으면 즉시 판별 가능. 커스텀 도메인이면 넘어가야 함.
https://myshop.wordpress.com → .com 확정
https://myshop.co.kr → 커스텀 도메인. 추가 확인 필요
2
wp-admin 접속 → 플러그인 메뉴 존재 여부
어드민 URL 뒤에 /wp-admin 붙여서 접속. 좌측 메뉴에 "플러그인" 항목 있는지 확인.
플러그인 메뉴 있음 → org 플러그인 없거나 업그레이드 유도 → com
접속 URL: https://도메인/wp-admin
3
wp-admin → 외모 > 테마 편집기 존재 여부
외모 메뉴 안에 "테마 파일 편집기"가 있으면 PHP 직접 수정 가능한 org. com은 이 메뉴 자체가 없거나 잠겨있음.
테마 편집기 있음 → org, 코딩 가능 없음 → com, 의뢰 거절 대상
4
FTP/SSH 계정 여부 클라이언트에게 직접 물어보기
가장 확실한 방법. FTP 계정이 있다는 건 자기 서버에 설치된 org라는 뜻. com은 FTP가 존재 자체를 안 함.
FTP/SSH 있음 → org 확정 FTP 없다 / 모른다 → com 가능성 높음
5
어느 호스팅 쓰는지 물어보기
국내 호스팅(카페24, 가비아, 닷홈, 클라우드웨이즈)이면 org 확정. "워드프레스닷컴" 이라고 하면 com.
카페24, 가비아, 닷홈, AWS, 클라우드웨이즈 → org "워드프레스 사이트에서 만들었어요" → com 의심
클라이언트에게 물어볼 스크립트

Yes → com 확정. 거절 No → 다음 확인
wp-admin에 플러그인 메뉴 있나?
Yes → org 유력 No → com. 거절
FTP 계정 있나?
Yes → org 확정. 의뢰 수락 가능 No → 호스팅 이름 물어보기
카페24 / 가비아 / 닷홈 / AWS 등 국내외 일반 호스팅?
Yes → org 확정. 의뢰 수락 모른다 / wordpress.com → 거절 또는 이전 요청
com 의뢰 받았을 때 거절 문구
클라이언트용 문구 예시
확인해보니 현재 WordPress.com 서비스를 이용 중이신 것 같아요. WordPress.com은 외부 개발자가 코드를 직접 수정할 수 없는 구조라, 원하시는 형태의 개발 작업이 기술적으로 불가능합니다. 두 가지 방법을 말씀드릴 수 있어요. 1. 카페24 같은 일반 호스팅으로 이전 후 작업 (비용/기간 별도 협의) 2. WordPress.com 비즈니스 플랜($25/월 이상) 업그레이드 후 제한적 작업 가능 어떻게 진행하실지 알려주시면 다시 안내드릴게요.
